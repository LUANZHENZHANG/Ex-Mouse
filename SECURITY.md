# 安全政策

## 支持范围

顺鼠目前处于早期开发阶段，只维护 `main` 分支上的最新版本。旧版本可能不会单独获得
安全修复。

## 报告安全问题

请不要在公开 Issue 中披露尚未修复的安全漏洞。

优先使用 GitHub 仓库的 **Security → Report a vulnerability** 私下提交报告：

<https://github.com/LUANZHENZHANG/macmouseplus/security/advisories/new>

如果该入口不可用，请创建一个不包含漏洞细节的 Issue，注明“需要私下报告安全问题”，
由维护者建立私密沟通渠道。

报告中建议包含：

- 受影响的版本或 commit
- 问题类型和潜在影响
- 可重复的验证步骤或最小示例
- 建议的修复方式（如有）

维护者会尽快确认报告、评估影响，并在修复可用后协调披露时间。

## 安全边界

顺鼠需要辅助功能、输入监控和自动化权限，并会处理系统级鼠标事件。这些权限能力较高，
请只运行自己构建或来自可信发布渠道的版本。

当前项目：

- 不连接网络或上传事件
- 不记录键盘输入
- 不包含自动更新功能
- 构建脚本使用临时代码签名，不提供 Apple 公证

任何新增联网、遥测、自动更新或输入记录行为的修改，都必须在代码审查和用户文档中明确说明。
